## Partner Folder Structure

Each partner has a dedicated folder containing:

```
partners/[partner-name]/
├── README.md           — Relationship summary and current status
├── COVER_LETTER.md     — Outreach or introduction letter (if applicable)
├── brief.md            — Internal partner brief (contacts, terms, health)
└── initiatives/        — Joint projects and agreements
```

For external-facing partner packages (e.g., presentations, one-pagers), these are also stored in the partner folder.

---

## Guidelines

- Each active partner has a folder with an up-to-date brief and status.
- Inactive or former partners are moved to `/archive/partners/` with date suffix.
- Partner packages (presentations, cover letters) are stored in the partner folder — not in `/marketing`.
- All partner agreements are indexed here but stored in secure external systems (no contracts with PII in-repo).
- Partnerships require clear value exchange, defined scope, and exit conditions — per [PRINCIPLES.md](../PRINCIPLES.md).
